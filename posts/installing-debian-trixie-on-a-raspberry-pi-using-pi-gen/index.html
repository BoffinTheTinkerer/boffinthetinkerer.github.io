Installing Debian Trixie on a Raspberry Pi using pi-gen

November 1, 2025 · 2 min · Boffin

This has got to be one of the easiest things I’ve ever done with a Pi.

I’ve just moved all of the web hosting to my SheevaPlug and wanted to update my Pi to Trixie.

I could wait until there is an official release or I could build my own.

I expected it to be a daunting process but it really came down to:

  • Install some dependencies

  • Clone a repo

  • Add some empty files to a couple of directories

  • Create a config file

  • Build

The instructions are pretty much cherry picked from:

https://github.com/RPi-Distro/pi-gen

Install the dependencies:

sudo apt install coreutils quilt parted qemu-user-static debootstrap zerofree zip \
dosfstools e2fsprogs libarchive-tools libcap2-bin grep rsync xz-utils file git curl bc \
gpg pigz xxd arch-test bmap-tools kmod

Clone the 64 bit repo:

git clone --branch arm64 https://github.com/RPI-Distro/pi-gen.git

Then, become root:

sudo su

And move into the pi-gen folder:

cd pi-gen

I’m interested in creating a ‘Lite’ version i.e. no desktop.

Create an empty file named SKIP in the directories stage3, stage4, and stage5

Create an empty file named SKIP_IMAGE in the directories stage4 and stage5

Create a config file:

nano config

The following will create an IMG file.

# Name of the image
IMG_NAME="raspios-trixie-arm64"

# Don't compress the image – output raw .img file
DEPLOY_COMPRESSION=none

(Remove DEPLOY_COMPRESSION=none for a ZIP output.)

There are a lot more config options than the two lines I used.

However, I kept it simple as:

  • The defaults are extremely sane

  • My next step is to use rpi-imager to write to an SSD and this allows for a few tweaks

Then, build:

./build.sh

Note: I built on the Pi before copying the IMG back to my laptop and running rpi-imager.

I had a couple of weird errors at the end but it writes to the SSD and boots into Trixie with no issues.

It really was that straightforward.

My Pi4 is in an Argon ONE M.2 SSD case and I think it took longer to dig out the weird USB-A to USB-A cable than the time it took to go through the process above.

Flashing to the Argon One SSD completed

Now, to configuration .

Edit: Naturally within a couple of weeks of my completing this, the Trixie version was available via rpi-imager.
